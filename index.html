    • Graph-explorer

    A graphite dashboard that's different, from Vimeo

    Structures your metrics

    The Graphite tree model is hard to organize, to query and lacks information. GE builds a multi-dimensional metric database by converting your string based metrics such as 

    servers.dfvimeodfs1.diskspace._srv_node_dfs10.byte_used
    into structured, tag-indexed metrics: 
    {
    server=dfvimeodfs1
    plugin=diskspace
    mountpoint=_srv_node_dfs10
    what=bytes
    type=used
    target_type=gauge
}

    Compose graphs on the fly

    Filter on any tag, using regexes, equality checks, negation, ...

    Everything has units

    Plugins yield one or more tag-based metrics for every string metric, and they get well-defined units. (for example a metric can be made available as a gauge in bytes, and as a rate in bytes/s)

    Group metrics into graphs

    Group by arbitrary tags. For example: show bytes used for all mountpoints on a graph per server, or a graph per mountpoint, listing how full that mountpoint is on different servers?

    Online aggregation

    Choose one or more tags to combine targets by, and represent them as one metric. For example: show for each server - on one graph - the sum of bytes used across all matching filesystems on that server.

    Interactive graphs

    realtime zooming, inspect datapoints, switch display mode, etc.

    And much more...
